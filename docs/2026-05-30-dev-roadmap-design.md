# Dev Roadmap — Design / Spec

**Дата:** 2026-05-30
**Автор:** Mikhail Logachev (Senior Backend Engineer)
**Статус:** Утверждён к реализации

---

## 1. Цель и философия

Вырасти из senior-разработчика продуктового аутсорса в **более глубокого и широкого инженера уровня strong senior → staff / principal / lead**.

**Ключевая установка:** рост ради роста, а не зубрёжка под собеседование. Качество интервью и знаний подтягивается как **побочный продукт** глубины. Поэтому система — устойчивая и ритмичная, а не хрупкий «90-дневный скрипт под дедлайн».

**Явно вне scope (out of scope):** подготовка к конкретным собеседованиям через мок-интервью, тестирование на время, контест-симуляции, заготовка ответов для HR. Эти механики исключены сознательно.

## 2. Вводные и ограничения

- **Бюджет времени:** ~1 час/день (иногда до 2 часов). Дизайн дневных заданий обязан помещаться в ~1 час без перегруза; антивыгорание встроено.
- **Целевой рынок (контекст, не цель):** российский бигтех (Yandex, T-Bank, Sber). Следствие: алгоритмы тяжёлые (особенно Yandex), system design, live coding. Английский и behavioral — легче, но не ноль.
- **Старт по DSA:** с нуля → нужен фундамент + ежедневные повторы (spaced repetition).
- **Старт по core (JS/TS/Node):** больше прикладного, чем внутренностей → начинаем «под капотом» системно.
- **Формат:** ротация треков + ежедневная алго-разминка.
- **Горизонт:** ≈13 недель (первый цикл). Это система, а не финишная прямая — цикл можно повторять/расширять.

## 3. Критерии успеха

- Ежедневная практика устойчива (серии/streak не рвутся системно), без выгорания.
- По каждому треку — заметный сдвиг глубины, зафиксированный в рефлексиях и `progress.json`.
- DSA: уверенное распознавание паттернов и решение medium-задач с проговариванием сложности.
- System Design: способность с нуля спроектировать и **защитить дизайн** через design doc.
- Капстоун-проект завершён с написанным design-документом.
- Прогресс durable и читаем (файл `progress.json`), пригоден для обратной связи от AI-коуча.

## 4. Треки (что учим)

### Трек 1 — Алгоритмы и структуры данных (DSA) — *ежедневный костяк*
Через **паттерны**, а не «500 задач наугад». Каждый паттерн = концепт + распознавание «когда применять» + 3–5 задач + spaced repetition.
- База: сложность O(), амортизация, массивы, строки, хеш-таблицы, стек/очередь, связные списки.
- Паттерны: два указателя, скользящее окно, префиксные суммы, бинарный поиск (и по ответу), монотонный стек/очередь.
- Деревья и графы: обходы, BFS/DFS, топосортировка, кучи/приоритетные очереди, union-find, trie.
- Парадигмы: рекурсия → backtracking → жадные → динамика (1D/2D/подпоследовательности/интервалы) → битовые операции.
- Навык объяснения: проговаривать решение и оценивать сложность на лету (мышление и коммуникация в код-ревью/дизайне, **не** репетиция интервью).

### Трек 2 — JavaScript вглубь
- Модель исполнения: контексты, scope chain, hoisting, TDZ, замыкания.
- Прототипы и наследование, `this`, `new`, `class` под капотом.
- Типы, приведение, equality, value vs reference, иммутабельность.
- Event loop: call stack, microtasks vs macrotasks, порядок выполнения.
- Промисы и async/await изнутри, генераторы, итераторы.
- Модули (CJS vs ESM), Symbols, Proxy/Reflect, WeakMap/WeakRef.
- Память и GC: mark-and-sweep, поколения, утечки и их поиск.

### Трек 3 — TypeScript вглубь
- Структурная типизация, generics (ограничения, дефолты, вывод).
- Union/intersection, литеральные типы, conditional types + `infer`, mapped types, key remapping, template literal types.
- Utility-типы (и как писать свои), `satisfies`, const-assertions.
- Вариантность (co/contra/bi), `unknown`/`never`/`any`, сужение и type guards, discriminated unions.
- Declaration files, module augmentation, строгость компилятора, производительность типов.

### Трек 4 — Node.js вглубь
- Архитектура рантайма: V8 + libuv + bindings.
- Фазы event loop в Node (timers/poll/check/close), thread pool, async I/O.
- Streams (все виды, backpressure, pipeline, object mode), Buffers/TypedArrays, EventEmitter.
- Многозадачность: worker_threads, cluster, child_process, graceful shutdown, сигналы.
- `async_hooks`/AsyncLocalStorage, обработка ошибок и подводные камни.
- Диагностика: perf_hooks, `--inspect`, heap snapshots, flame graphs, поиск утечек, профилирование под нагрузкой.

### Трек 5 — System Design
- Фундамент: сбор требований, оценка «на салфетке», числа латентности, CAP/PACELC.
- Строительные блоки: балансировщики, кэши, CDN, SQL/NoSQL, очереди, поиск, объектное хранилище.
- Масштабирование: репликация, шардирование/партиционирование, consistent hashing, read/write пути.
- Надёжность: redundancy, failover, rate limiting, circuit breaker, идемпотентность, backpressure.
- Кейсы: rate limiter, лента новостей, чат/мессенджер, нотификации, шортенер, поиск-автокомплит, платежи — и собственный activity-timeline из Atlas как разбор кейса.
- Навык: как формулировать и защищать дизайн — design doc, ведение технической дискуссии (staff-навык, **не** интервью-раунд).

### Трек 6 — Паттерны и архитектура
- SOLID, GoF (порождающие/структурные/поведенческие) — на идиоматичном TS/Node.
- DI/IoC, Repository/Unit of Work.
- Архитектурные: слоистая, гексагональная (ports/adapters), Clean, DDD (тактический + стратегический), CQRS, Event Sourcing, Saga, Outbox (есть практика — добиваем теорию).
- Анти-паттерны, code smells, техники рефакторинга, ADR.

### Трек 7 — Распределённые системы (теория)
- Модели отказов, «8 заблуждений распределённых систем».
- Время и порядок: логические/векторные часы, Lamport.
- Модели согласованности: линеаризуемость, последовательная, причинная, eventual.
- Репликация (leader/follower, multi-leader, leaderless, кворумы), партиционирование.
- Консенсус: Raft вглубь (+ интуиция Paxos).
- Распределённые транзакции (2PC, саги), семантики доставки сообщений, «иллюзия exactly-once», CRDT (обзорно).

### Трек 8 — Базы данных вглубь (PostgreSQL)
- Хранение и MVCC, мёртвые кортежи, VACUUM, WAL, чекпойнты.
- Индексы: B-tree, Hash, GIN, GiST, BRIN, частичные, покрывающие, по выражению.
- Планировщик и `EXPLAIN (ANALYZE, BUFFERS)`, статистика, алгоритмы join (nested loop/hash/merge).
- Уровни изоляции и аномалии, блокировки (row/table, advisory, `FOR UPDATE SKIP LOCKED`), deadlocks.
- Репликация (streaming/logical), партиционирование, шардирование, пулинг (pgbouncer), тюнинг.

### Трек 9 — CS-фундамент (сети / ОС / конкурентность)
- Сети: TCP/IP, TCP (handshake, flow/congestion control), UDP, DNS, HTTP/1.1 → HTTP/2 → HTTP/3/QUIC, TLS-handshake, WebSockets, gRPC, L4/L7-балансировка.
- ОС: процессы vs потоки, планировщик, контекстное переключение, виртуальная память, syscalls, file descriptors, модели I/O (epoll/kqueue → связь с libuv), сигналы, IPC.
- Конкурентность: гонки, мьютексы/семафоры, deadlocks, атомарность, модели памяти, async vs parallel, actor/CSP vs однопоточный event loop.

### Тонкие нити (лёгкие, по ходу)
- **Observability/SRE:** 3 столпа (метрики/логи/трейсы), RED/USE, SLI/SLO/error budget, OpenTelemetry, постмортемы.
- **Security:** OAuth2/OIDC, JWT, OWASP Top 10, prototype pollution, секреты, крипто-основы.
- **English (опц.):** проговаривание system design на английском, технический сторитейлинг — 1 лёгкий блок/нед.
- **Behavioral / staff (тонкая нить):** STAR, staff-архетипы (tech lead / architect / solver / right-hand), влияние без власти, design docs/RFC — 1 рефлексия/нед как осмысление роста, мапим реальный опыт Atlas в staff-нарративы.

## 6. Движок дня (как выглядит ~1 час)

Недельный шаблон: **6 рабочих дней + 1 разгрузочный**. Каждый день: **алго-разминка 15–20 мин** + **тема дня 30–40 мин** + **рефлексия 5 мин** (фиксируется в трекере).

| День | Тема дня (30–40 мин) |
|---|---|
| Пн | Алгоритмы (глубокая сессия) — паттерн недели + 2–3 задачи |
| Вт | Core-язык A — JS / TS / Node (по фазе) |
| Ср | System Design |
| Чт | Core-язык B — JS / TS / Node (по фазе) |
| Пт | Фундамент — распределённые / БД / CS-фундамент (ротация по неделям) |
| Сб | Паттерны/архитектура + мини-интеграция + behavioral-рефлексия |
| Вс | Разгрузка / нагон / spaced-repetition повторы |

Веса: core-язык — 2 дня/нед (приоритет); алгоритмы — ежедневная разминка + глубокий понедельник (перекос под РФ-бигтех); фундаментальные треки — ~1 день/нед с фазовым акцентом.

**Spaced repetition (DSA):** решённые паттерны/задачи попадают в очередь повторов с возрастающими интервалами; разгрузочный день и разминки используют эту очередь.

## 7. Архитектура приложения (трекер)

**Решение:** один самодостаточный HTML-файл + `localStorage` (автосейв) + экспорт/импорт в `progress.json`.

Обоснование: открывается двойным кликом, без сервера и сборки, работает офлайн. `fetch` из `file://` блокируется браузером (отвергнуто — лишнее трение); фреймворк со сборкой — оверкилл (против простоты).

**Компоненты (каждый с одной ответственностью):**
- `index.html` — разметка + стили + логика приложения (рендер, состояние, взаимодействие).
- `curriculum.js` — данные учебной программы (фазы, недели, дни, треки, задачи). Грузится через `<script src>` (работает на `file://`). Легко редактируется отдельно от логики.
- `progress.json` — экспортируемое состояние прогресса («память в файле»): отметки выполнения, заметки/рефлексии, серии, очередь spaced repetition. Durable, портативно, читаемо AI-коучем.

**Поток данных:** `curriculum.js` (статичная программа) + состояние из `localStorage` → рендер дашборда и карточки «сегодня» → действия пользователя (чек-боксы, заметки) → автосейв в `localStorage` → по кнопке экспорт в `progress.json` (и импорт обратно).

**Возможности трекера:**
- Дашборд прогресса по фазам и трекам.
- Карточка «сегодня»: задачи дня с чек-боксами + поле рефлексии.
- Серии/streak.
- Прогресс по DSA-паттернам с очередью spaced repetition.
- Экспорт/импорт `progress.json`.
- Тёмная тема, полностью офлайн.

**Обработка ошибок:** битый/отсутствующий `localStorage` → старт с чистого состояния без падения; импорт невалидного `progress.json` → понятное сообщение, текущее состояние не затирается.
